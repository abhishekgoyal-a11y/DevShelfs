Prompt Templates

PromptTemplate, ChatPromptTemplate, few-shot prompts, structured outputs, guardrails, and debugging tips.

Series progress3 / 8

PromptTemplate

A PromptTemplate is a string with named placeholders. You fill them in at runtime.

from langchain_core.prompts import PromptTemplate template = PromptTemplate.from_template( "You are a helpful assistant. Answer this question: {question}" ) prompt = template.invoke({"question": "What is LangChain?"})

Use PromptTemplate when working with plain LLMs (text-in). For chat models, use ChatPromptTemplate.

ChatPromptTemplate

Chat models expect a list of messages. ChatPromptTemplate lets you template that list with variables.

from langchain_core.prompts import ChatPromptTemplate prompt = ChatPromptTemplate.from_messages([ ("system", "You are an expert in {topic}. Be concise."), ("human", "{question}"), ]) chain = prompt | llm | StrOutputParser() chain.invoke({"topic": "Python", "question": "What is a decorator?"})

The system message sets the context; the human message is the user's input. Both accept variables.

Few-shot prompts

Show the model examples of the input/output pattern you expect. This significantly improves output quality for structured tasks.

from langchain_core.prompts import FewShotChatMessagePromptTemplate examples = [ {"input": "happy", "output": "sad"}, {"input": "tall", "output": "short"}, ] example_prompt = ChatPromptTemplate.from_messages([ ("human", "{input}"), ("ai", "{output}") ]) few_shot_prompt = FewShotChatMessagePromptTemplate( examples=examples, example_prompt=example_prompt, ) final_prompt = ChatPromptTemplate.from_messages([ ("system", "Give the antonym of the word."), few_shot_prompt, ("human", "{input}"), ])

Structured outputs (JSON mode)

Instead of parsing free text, you can tell the model to return a specific JSON schema every time. LangChain's .with_structured_output() handles this cleanly using Pydantic.

from pydantic import BaseModel from langchain_openai import ChatOpenAI class MovieReview(BaseModel): title: str rating: int # 1–10 summary: str llm = ChatOpenAI(model="gpt-4o-mini") structured_llm = llm.with_structured_output(MovieReview) result = structured_llm.invoke("Review the movie Inception") print(result.title) # "Inception" print(result.rating) # 9 print(result.summary) # "A mind-bending thriller..."

The model is forced to output valid JSON matching your Pydantic schema. No output parsing needed — you get a typed Python object back.

Guardrails & validation

Structured outputs give you implicit validation — if the model's output does not match the schema, LangChain raises an error. For explicit guardrails, add validators to your Pydantic model:

from pydantic import BaseModel, field_validator class MovieReview(BaseModel): title: str rating: int summary: str @field_validator("rating") @classmethod def rating_in_range(cls, v): if not 1 <= v <= 10: raise ValueError("Rating must be between 1 and 10") return v

If the model returns a rating of 11, the validator rejects it before it reaches your code. Combine with LangChain's .with_retry() to automatically ask the model to try again.

Prompt debugging tips

1. Print the rendered prompt before sending

print(prompt.invoke({"topic": "Python", "question": "..."}))

Always verify the prompt looks exactly as intended before debugging the model's output.

2. Enable verbose mode

from langchain.globals import set_verbose
set_verbose(True)

Prints every prompt and response to the console as they happen. Turn off in production.

3. Use LangSmith for tracing

Set LANGCHAIN_TRACING_V2=true and LANGCHAIN_API_KEY to get a full trace of every chain run in the LangSmith UI — inputs, outputs, latency, token counts.

4. Test prompts in isolation

Before adding a prompt to a chain, invoke it standalone with llm.invoke(prompt.invoke({...})). Isolate whether the issue is the prompt or the chain.

Quick summary

  • Use ChatPromptTemplate for all modern chat models
  • Few-shot examples dramatically improve output quality for structured tasks
  • .with_structured_output(PydanticModel) returns typed Python objects, no parsing needed
  • Add field_validator to enforce output constraints and combine with .with_retry()
  • Debug with set_verbose(True) locally and LangSmith in production
