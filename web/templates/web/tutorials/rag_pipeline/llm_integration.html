Prompt Engineering for RAG

Bad RAG Prompt

"Use these documents to answer: {docs}
Question: {query}
Answer:"

Problems: Ambiguous instructions. LLM doesn't know to cite sources.
Result: Hallucinations, made-up citations.

✓ Better RAG Prompt

You are a helpful assistant. Your job is to answer questions using ONLY
the provided documents. If the answer is not in the documents, say:
"I don't have this information in the available documents."

Always cite your sources using the [Source: doc_title] format.

{DOCUMENTS}

Question: {QUERY}

Answer: Let me search the documents...
[Your answer with citations]

Prompt Tips:

  • Be explicit about using only provided docs
  • Define what to do when info isn't available
  • Show citation format with examples
  • Use lower temperature (0.3-0.5) for consistency
  • Add examples in few-shot format

    if not citations:
        return False, "No citations found"

Chain Patterns for Long Documents

Stuff (Simple)

Put all docs in context, ask LLM to answer. Best for <10K tokens.

Map-Reduce (For parallel)

Summarize each doc separately, then combine summaries. Works for large datasets.

Refine (Iterative)

Start with first doc, iteratively refine with next docs. Best quality but slow.

Recommendation: Use "Stuff" for RAG. If docs exceed context, retrieve better quality docs with reranking instead of using chains.
