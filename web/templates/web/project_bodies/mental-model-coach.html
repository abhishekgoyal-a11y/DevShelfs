Mental Model & Learning Coach is a beginner-friendly project that brings structured thinking frameworks to real problems. Choose from three modes: apply a framework to your own situation, take a lesson on any mental model, or get guided journaling prompts to process what’s on your mind.

Twelve frameworks are built in — from First Principles and Inversion to Systems Thinking and the Feynman Technique. Each is described with a one-sentence prompt instruction that gets injected into the system message at generation time, shaping the analysis without duplicating logic. The reflection mode generates depth-scaled journaling prompts (3 for a quick check-in, up to 10 for a deep session) and provides a text area to write directly in the app.

Built with Python, Streamlit, and the Groq API, this project shows how a single app can serve multiple use cases through mode switching and data-driven prompt construction.

Purpose: practice data-driven prompt design — where the prompt content is assembled from a dictionary rather than hardcoded, making it trivial to add new frameworks.

Typical use: facing a hard career decision? Select “Apply a Framework,” choose Inversion, describe the situation, and get a structured failure-mode analysis with recommended next actions.

Key ideas you will touch:

  • Dictionary-driven prompts — twelve frameworks stored as name → description pairs; the selected description is inserted into the system message automatically.
  • Mode switching with st.radio — three modes on one page without routing or separate files.
  • Depth scaling — a select_slider changes the number of reflection prompts requested; the instruction string adapts.
  • In-app journaling — a text area below generated prompts lets users write and download their entry without switching apps.

Overall flow

User selects mode via radio button
      ↓
Mode 1 — Apply Framework:
  Select model + describe problem → Groq applies framework step-by-step + recommended actions + reflection Qs
      ↓
Mode 2 — Learn a Model:
  Select model + optional domain → Groq teaches with analogies, examples, steps, and a practice exercise
      ↓
Mode 3 — Reflective Journal:
  Describe situation + pick depth → Groq generates prompts + suggests frameworks + offers insight
  → In-app text area to write + download journal entry

Step-by-Step Implementation

  1. Set up the project
    • Install Python 3.10+, create folder and virtual environment
  2. Install dependencies
    • streamlit, groq, httpx, truststore
  3. Build the mental models dictionary
    • A Python dict maps each framework name to a one-sentence description; this drives both the selectbox labels and the prompt injection
  4. Implement mode 1 — Apply Framework
    • System message: inject framework name + description, specify six output sections (explanation, analysis, insights, actions, reflection questions)
    • Download button for the analysis
  5. Implement mode 2 — Learn a Model
    • System message: teacher persona, seven sections including an analogy, step-by-step application, common mistakes, and a quick practice exercise
  6. Implement mode 3 — Reflective Journal
    • st.select_slider maps to prompt count instruction strings
    • After rendering prompts, show a text area + download button for the journal entry

📖 How the Code Works (Step-by-Step)

1. Dictionary-driven prompt injection

  • MENTAL_MODELS[selected_model] retrieves the one-sentence description and injects it into the system message at runtime — adding a new framework requires only one line in the dictionary.

2. Mode switching without routing

  • st.radio returns a string; three if/elif blocks render the appropriate UI and prompt logic. No page files or URL routing needed.

3. Depth-scaled reflection prompts

  • A depth_map dict maps slider values to instruction strings (“3 quick questions” vs “8-10 multi-layered questions”); the selected string is dropped into the system message — no branching logic around question count.

4. In-app journaling with download

  • The journal text area and download button are rendered conditionally only after the prompts are generated, so the UI stays focused without empty widgets at startup.
  • The download includes both the original situation and the reflection text — providing useful context when re-reading the entry later.
