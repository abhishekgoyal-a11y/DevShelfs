Travel Planner Agent is a beginner-friendly project that generates a complete, personalized travel itinerary from a single form. Enter your destination, budget per day, travel style, interests, and any preferences to avoid — the agent returns a day-by-day plan with hotel recommendations, food picks, transport options, packing tips, and money-saving advice.

Seven travel styles are available — from Budget Backpacker to Luxury to Digital Nomad — and each is injected into the system prompt so the output adapts in vocabulary, accommodation tier, and activity choices. A follow-up chat below the itinerary lets you refine any part of the plan or ask about visa requirements, local customs, or alternative routes.

Built with Python, Streamlit, and the Groq API, this project demonstrates form-to-prompt design and lightweight chat augmentation of a generated document.

Purpose: practice translating structured form inputs into a rich, multi-section prompt — a pattern used in every AI generation tool that takes user preferences.

Typical use: plan a 7-day Tokyo trip on a $120/day mid-range budget with food and history interests, download the itinerary, then ask the chat “Can you suggest a day trip to Nikko?”

Key ideas you will touch:

  • st.form — groups all inputs and submits them as one batch, preventing Streamlit from re-running on every keystroke.
  • Rich prompt assembly — destination, days, budget, style, pace, interests, and special notes assembled into a structured user message.
  • Multi-section output format — system message specifies nine output sections with consistent headings for predictable rendering.
  • Follow-up chat with context — the form values are summarized as context for the chat system message so follow-up answers stay trip-specific.

Overall flow

User fills in form: destination, days, budget, style, interests, notes
      ↓
Clicks "Plan My Trip"
      ↓
Groq generates day-by-day itinerary + accommodation + food + transport + packing + tips
      ↓
Itinerary displayed + Download button
      ↓
Follow-up chat: user refines or asks questions
  → trip context injected as summary in system message
  → Groq answers with trip-specific details

Step-by-Step Implementation

  1. Set up the project
    • Install Python 3.10+, create folder and virtual environment
  2. Install dependencies
    • streamlit, groq, httpx, truststore
  3. Build the form
    • Wrap all inputs in st.form to batch submit
    • Two-column layout for paired fields: destination / origin, days / budget, style / pace
    • st.multiselect for interests, free-text for avoid and special notes
  4. Write the itinerary prompt
    • System message specifies 9 output sections: Trip Overview, Day-by-Day (with morning/afternoon/evening and daily cost), Accommodation, Food Guide, Getting Around, Packing, Money Tips
    • User message assembles all form values as labeled fields
  5. Add the follow-up chat
    • Chat system message includes a one-line trip context summary built from form values so answers stay relevant without repeating the full prompt
  6. Add download button
    • st.download_button with file_name derived from the destination name

📖 How the Code Works (Step-by-Step)

1. st.form prevents premature re-runs

  • Wrapping inputs in st.form means Streamlit only re-runs the script when “Plan My Trip” is clicked, not on every keystroke — preventing accidental API calls mid-typing.

2. Nine-section output format

  • Each section in the system message is prefixed with ## so st.markdown() renders them as consistent H2 headings across different response lengths.
  • The daily cost line inside each day block gives the user a quick budget reality check without a separate calculation.

3. Conditional prompt assembly

  • Origin, avoid, and special notes are appended to the user message only when non-empty — keeping the prompt clean and the model from filling in those fields with assumptions.
