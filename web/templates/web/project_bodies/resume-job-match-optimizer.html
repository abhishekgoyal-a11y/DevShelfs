Resume + Job Match Optimizer is an intermediate-level project that transforms job applications into a data-driven workflow. Upload a resume (or paste the text) alongside a job description, then choose from three actions: score the fit, get structured improvement suggestions, or generate a tailored cover letter.

The fit scorer returns a 0–100 match score with a breakdown of present keywords, missing keywords, top strengths, and top gaps. The improvement flow forces the model to produce exactly four labeled sections including rewritten bullet points in STAR format. The cover letter path generates a structured 3-4 paragraph letter and offers a download button so you can send it immediately.

Built with Python, Streamlit, Groq API, and pypdf, this project demonstrates how structured prompting and multi-action UI design turn a single LLM call into a focused career tool.

Purpose: practice structured prompting and multi-button Streamlit layouts that branch into distinct LLM calls from a shared input.

Typical use: paste your resume, paste a job posting, click “Score Fit & Analyze” to see the keyword gap, then “Generate Cover Letter” and download the draft.

Key ideas you will touch:

  • PDF extraction — pypdf reads uploaded resumes without any cloud parser.
  • Fit scoring — a structured system message that forces a numeric score, keyword lists, and gap analysis in one response.
  • Three-button branching — shared validation then separate prompt logic per action, avoiding duplicated API calls.
  • Download button — st.download_button streams the cover letter as a .txt file with no server storage.

Step-by-Step Implementation

  1. Set up the project
    • Install Python 3.10+, create a folder and virtual environment
  2. Install dependencies
    • streamlit, groq, httpx, truststore, pypdf
  3. Store your API key
    • GROQ_API_KEY in .streamlit/secrets.toml
  4. Build the two-column layout
    • Left column: PDF uploader + text area pre-filled with extracted text
    • Right column: job description text area
    • Three action buttons in a row below both columns
  5. Extract PDF text
    • Wrap uploaded bytes in io.BytesIO, pass to pypdf.PdfReader, concatenate all pages
    • Cap at MAX_RESUME_CHARS to keep prompts manageable
  6. Craft the three prompts
    • Score: system message forces numeric score, keyword lists, strengths, and gaps in labeled sections
    • Improve: system message forces four sections including STAR-rewritten bullet examples and ATS keywords
    • Cover letter: specifies hook, experience match, achievements with numbers, and call-to-action — output letter text only
  7. Display and download
    • All outputs render via st.markdown()
    • Cover letter adds st.download_button with mime="text/plain"
  8. Guardrails
    • Resume always required; job description required only for score and cover letter
    • Validation runs once before any API call to avoid redundant checks

📖 How the Code Works (Step-by-Step)

1. Two-column input layout

  • st.columns(2) places the resume and job description panels side-by-side on wide screens and stacks them on mobile.
  • The resume text area is pre-filled with extracted PDF text, so users can edit the content before sending it to the model.

2. Shared validation before branching

  • One if do_score or do_improve or do_cover: block runs shared checks so you don’t repeat the same resume-empty guard in three places.
  • The job description gate fires only for the paths that require it, keeping the improve flow usable without a job posting.

3. Fit score prompt design

  • The system message specifies every section heading and format (e.g., ## Fit Score: [0-100]/100), preventing the model from producing a narrative paragraph instead of the structured output the UI expects.
  • Both resume and job description are injected as labeled blocks in the user message so the model treats them as separate inputs.

4. Cover letter prompt

  • The system message explicitly says “output only the letter text — no subject line, no meta-commentary” to stop the model prefixing “Here is your cover letter:”.
  • st.download_button passes the string directly as data—no temp file, no server storage.
