Dify: The Open-Source Platform for Building LLM Apps Visually

Dify is an open-source LLM application development platform that lets you build chatbots, RAG pipelines, and AI agents through a visual workflow editor—with built-in observability and one-click deployment.

Try Dify ↗ GitHub ↗ Docs ↗

What is Dify?

Dify (formerly known as LangGenius) is an open-source platform for building, deploying, and operating LLM-powered applications. Instead of writing orchestration code from scratch, you build your app visually—connecting LLMs, data sources, tools, and logic in a drag-and-drop workflow editor.

It supports every major LLM provider (OpenAI, Anthropic, Mistral, Llama, and more), has a built-in RAG pipeline for knowledge bases, and ships with observability tools so you can monitor token usage, latency, and quality out of the box. You can use the hosted cloud version or self-host with Docker.

Key features of Dify

1. Visual workflow editor

Build complex LLM pipelines by connecting nodes—LLM call, document retrieval, code execution, HTTP request, conditional logic—without writing orchestration code.

2. Built-in RAG pipeline

Upload documents (PDFs, Word files, web pages, Notion pages) to create a knowledge base. Dify handles chunking, embedding, vector storage, and retrieval—you connect it to your workflow with a single node.

3. Multi-model support

Connect OpenAI, Anthropic Claude, Google Gemini, Mistral, local Ollama models, and many more—all from the same interface. Switch models without rewriting your pipeline.

4. AI agents with tools

Build ReAct-style agents that can use tools—web search, code execution, API calls, database queries—through a visual agent configuration panel rather than code.

5. Observability and analytics

Every conversation and workflow run is logged with token usage, latency, user feedback, and model responses—giving you the data to evaluate and improve your application.

6. One-click embed and API

Publish your app as a shareable chatbot, embed it on a website with a script tag, or consume it via API—without separate deployment infrastructure.

How Dify works

  • Create a new app (chatbot, workflow, or agent) from the dashboard
  • Connect your LLM provider API keys in settings
  • Build your pipeline visually—add knowledge bases, tools, and logic nodes
  • Test it in the built-in playground, then publish to a shareable URL or your API
  • Monitor usage and quality via the built-in logs and analytics

Real-life use cases of Dify

Internal knowledge base chatbot

  • Upload your company docs, HR policies, and SOPs and deploy a chatbot that answers employee questions accurately

Customer support automation

  • Build a RAG-powered support bot that answers from your product documentation with citations

Automated content pipelines

  • Build a workflow that takes a URL, summarises the content, categorises it, and writes a social post—all in one triggered pipeline

Pros and cons of Dify

Pros

  • Open-source and self-hostable
  • Visual workflow editor—no orchestration code
  • Built-in RAG, observability, and multi-model support
  • Active community with 40K+ GitHub stars
  • Works with local models via Ollama

Cons

  • Self-hosting requires Docker setup and some maintenance
  • Visual workflows can get complex on large applications
  • Less code-level flexibility than pure LangChain/LlamaIndex
  • Enterprise features require the paid cloud plan

Dify pricing

  • Community Edition (free) — fully featured, self-hosted via Docker
  • Cloud Free — hosted version with 200 message credits/month
  • Professional ($59/month) — 5,000 message credits, priority support
  • Enterprise (custom) — SSO, custom model deployment, SLA

Alternatives to Dify

  • Flowise — simpler drag-and-drop LLM flow builder, LangChain-based
  • LangChain — more code-level control, steeper learning curve
  • LlamaIndex — RAG-focused framework for Python developers
  • n8n — general workflow automation that also supports AI nodes

Who should use Dify?

  • Teams who want to build LLM apps without deep ML engineering expertise
  • Companies that need to self-host AI infrastructure for data privacy
  • Developers who want observability and RAG out of the box
  • Product teams that want to prototype and iterate on AI features quickly

Bonus section

Getting started with self-hosted Dify

  • Clone the repo: git clone https://github.com/langgenius/dify
  • Run cd dify/docker && docker compose up -d
  • Open localhost:80 and add your LLM API keys in settings

Dify is the fastest path from "I want an AI chatbot over my documents" to a production-ready, observable, self-hosted system—without writing orchestration logic from scratch.
